## 📝 Descripción del Proyecto
Aplicación móvil diseñada para una Agencia de Viajes que permite gestionar un catálogo de destinos turísticos mediante servicios de Firebase.

### 🚀 Funcionalidades y Requerimientos
* **Autenticación:** Sistema de Login y Registro de usuarios mediante Firebase Auth.
* **Base de Datos:** Almacenamiento de datos en la nube con Firebase Firestore.
* **Multimedia:** Uso de Firebase Storage para la carga y recuperación de imágenes de destinos.
* **Gestión de Catálogo (CRUD):** * **Create:** Registro de destinos con nombre, país (Spinner), precio y descripción (mínimo 20 caracteres).
    * **Read:** Visualización mediante RecyclerView con CardView y carga de imágenes optimizada.
    * **Update:** Edición completa de paquetes turísticos e imágenes.
    * **Delete:** Eliminación de destinos con diálogo de confirmación previa.

### 🛠️ Validaciones Obligatorias
* No se permiten campos nulos o vacíos.
* El precio debe ser obligatoriamente mayor a 0.
* Cada destino debe tener una imagen asociada para ser guardado.
* Visualización de errores y advertencias mediante Toasts personalizados.
